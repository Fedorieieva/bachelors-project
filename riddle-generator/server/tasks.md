# 🎓 Дипломний Проект: Веб-застосунок для Генерації Загадок за допомогою ШІ

## 🎯 Загальна Мета

Повна реалізація масштабованого бекенду (NestJS) та базового фронтенду для генерації персоналізованих загадок за допомогою Gemini API з елементами соціальних мереж (лайки, коментарі).

---

## 🏗️ Етап 1: Основа Проекту та База Даних (PostgreSQL/Prisma)

### 1.1. Налаштування Середовища
* Створити новий проект NestJS (використовуючи Nest CLI).
* Налаштувати модульну структуру, дотримуючись NestJS best practices.
* Налаштувати конфігурацію (модуль `Config`) для роботи зі змінними середовища (.env).

### 1.2. База Даних та Схема
* Налаштувати підключення до PostgreSQL.
* Інтегрувати Prisma ORM.
* Виконати міграцію бази даних, використовуючи надану `schema.prisma` (моделі `User`, `Riddles`, `Comment`, `Like` та `Session`).

### 1.3. Анонімна Аутентифікація (Anonymous Auth)
* Реалізувати логіку в модулі `Auth` для генерації унікального ID (`user.id`) та JWT-токена для нових анонімних користувачів.
* Реалізувати Guard для захисту маршрутів, який перевіряє валідність токена та ідентифікує користувача (`req.user.id`).

---

## 🤖 Етап 2: Інтеграція Генерації ШІ (Gemini API)

### 2.1. Налаштування та Конфігурація AI
* Встановити необхідний SDK для роботи з Gemini API (наприклад, `@google/genai`).
* Налаштувати сервіс `AiService` для безпечного використання `GEMINI_API_KEY` зі змінних середовища.

### 2.2. Сервіс Генерації Загадок
* Розробити DTO (Data Transfer Object) для вхідних параметрів генерації (наприклад, `topic: string`, `complexity: number`).
* Створити функцію `generateRiddle(params: RiddleDto)` у `AiService`.
* Скласти деталізований промпт для моделі Gemini, що вимагає:
  * Генерацію самої загадки.
  * Генерацію чіткої відповіді.
  * Виведення результату у стандартизованому JSON-форматі для легкого парсингу.

### 2.3. Збереження Генерації
* Реалізувати логіку в контролері `RiddlesController`, яка:
  * Викликає `AiService.generateRiddle`.
  * Зберігає отриману загадку, відповідь та контекст (`prompt_context`) у таблицю `Riddles`.
  * Присвоює загадку поточному автентифікованому користувачу (`authorId`).

---

## 🧩 Етап 3: Робота з Контентом та Стрічка

### 3.1. CRUD для Загадок
* Створити ендпоінти для:
  * `GET /riddles/:id` (читання загадки за ID).
  * `DELETE /riddles/:id` (видалення загадки, дозволено лише автору).
  * `PATCH /riddles/:id` (оновлення, наприклад, встановлення `is_public`).

### 3.2. Стрічка Публічних Загадок (Feed)
* Розробити ендпоінт `GET /feed`.
* Реалізувати логіку пагінації (limit, offset або cursor-based).
* Фільтрувати: показувати лише загадки, де `is_public` = `true`.
* Реалізувати сортування за: датою створення (`created_at`) та популярністю (`likes_count`).

### 3.3. Персональний Кабінет
* Створити ендпоінт `GET /users/me/riddles`.
* Показувати користувачу всі його загадки (як публічні, так і приватні).

---

## 💬 Етап 4: Соціальні Елементи

### 4.1. Лайки (Likes)
* Розробити ендпоінт `POST /riddles/:id/like`.
* Реалізувати функцію, яка:
  * Додає запис у таблицю `Like` (якщо його ще немає).
  * Видаляє запис (якщо він вже існує — "toggle like").
  * Атомарно оновлює лічильник `likes_count` у таблиці `Riddles`.

### 4.2. Коментарі (Comments)
* Розробити ендпоінт `POST /riddles/:id/comment` для додавання нового коментаря.
* Розробити ендпоінт `GET /riddles/:id/comments` для отримання списку коментарів до загадки.
* Забезпечити, щоб при додаванні коментаря оновлювався лічильник `comments_count`.
